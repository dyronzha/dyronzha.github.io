Google 套件練習

  1. 作品集_個人學習
  2. Google 套件練習
  3. 2018/07 ~ 2018/12

研究室新購一隻 Ar 手機,教授讓我研究看看,就想說能不能使用Google Ar Core做看看遊戲，構想是在手機偵測平面後可以召喚BOSS進行戰鬥，並利用自身移動來閃躲和用手在螢幕滑動做出反擊

想使用Firebase做現有的資料庫，學習相關的API看能不能做一款遊戲

學習重點

  • Google AR Core 遊戲想法測試
    • 分析官方給的範例和查閱 API Reference，能快速做出找到平面的 功能，比較要注意的是已偵測到平面的釋放和保留
    • 遊戲裡有功能是要依照劃的方向將子彈打回去，自己作法是抓碰到 球的觸控點和前後幾個點做平均向量，和自己所面向的向量做比例 相加，最後再丟回給子彈讓它飛向那個方向
    • 試著讓子彈都不掛上物理和 monobehavior 腳本，由一個管理者統 一更新和計算碰撞
    • 本來是想做出那種需要不斷移動和反擊的遊戲，但結果可能是因 為彈幕設計不完善和反擊的平衡度沒抓好，讓玩家只會偏好在原 地做反擊，不會移動
  • Google Firebase 連線學習測試
    • 分析官方給的範例和查閱 API Reference，能做出創建會員、登入 和上傳下載資料的功能
    • Firebase 所提供的是一種實時資料庫，供開發者上傳下載資料並在 儲存儲存成 Json 格式
    • 因為是利用第三方的資料庫，無法主動要求伺服器將更新資料推給 所有會員，所以使用的方式是利用 Firebase 所提供一種資料變更的 callback 來讓會員偵測如有資料變動就更新仔入資料
